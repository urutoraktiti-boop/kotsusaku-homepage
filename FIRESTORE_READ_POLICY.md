# Firestore読み取りルール

このリポジトリでは、Firestoreの無料枠と利用料金を守るため、統計表示の読み取り方を固定します。

## 必ず守ること

- ホームページは、初回表示時に `analytics_summary/summary` を1回読む。
- その後の自動更新は、毎時の集計反映後だけにする。
- 現在の基準は「毎時00分の少し後」、実装上は毎時 + 2分。
- タブが非表示の間は自動更新を止める。
- 表示する統計値は、サーバー側で集計済みの `analytics_summary/summary` を信頼する。

## 禁止すること

- `analytics` コレクション全件を、ホームページから直接読むこと。
- `db.collection('analytics').get()` をホームページに追加すること。
- `onSnapshot` で統計値を常時購読すること。
- 1分ごと、5分ごとなど、毎時より短い間隔で統計値を自動ポーリングすること。
- ユーザー端末から `analytics_summary/summary` を直接加算・再集計すること。

## 理由

`analytics` コレクションを全件読むと、ユーザー数に比例してFirestore readsが増えます。

例: ユーザーが1,000人いる場合、1回の全件読み取りで約1,000 readsになります。これを公開ページで行うと、少数のアクセスでも無料枠を超えやすくなります。

統計値はCloud Functionsなどのサーバー側集計で作り、ホームページは集計済みの1ドキュメントだけを読む方針にします。

## 変更前に確認すること

統計表示やFirestore読み取りを変更する前に、以下を確認してください。

```bash
rg -n "collection\\(['\\\"]analytics['\\\"]\\)\\.get\\(|onSnapshot|STATS_POLL_INTERVAL_MS|setInterval\\(_fetchStatsOnce" index.html
```

この検索で、公開ページからの全件読み取り・リアルタイム購読・短間隔ポーリングが増えていないことを確認します。
